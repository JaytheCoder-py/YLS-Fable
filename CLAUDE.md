# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

A responsive replica of the anthropic.com homepage (as of July 2026) plus a `/use-cases` page replicating the grid of claude.com/resources/use-cases, built with React 18 + Vite 6 + Tailwind v4 + GSAP. No tests and no linter are configured.

## Commands

```bash
npm install
npm run dev        # dev server on http://localhost:5173 (port fixed in vite.config.js)
npm run build      # production build into dist/
npm run preview    # serve the production build
```

Note: `npm run capture:ref` / `capture:build` reference `capture.mjs`, which is **not present** in the repo (along with `DESIGN.md`, `theme.css`, `tokens.json`, `variables.css`, which the README mentions). Those scripts will fail until the file is restored.

## Architecture

**Styling lives in CSS, not in JSX utility classes.** `src/index.css` holds:
- Design tokens in a Tailwind v4 `@theme` block (no `tailwind.config.js`) — colors like `--color-canvas` (#faf9f5), `--color-ink` (#141413), `--color-kraft` (#f5e3c7), and `--container-page: 1272px`. Tokens were measured from the live site; change them here, not inline.
- A handwritten `@layer components` section with BEM-ish class names (`.nav__link`, `.vcard__eyebrow`, …). Components reference these classes rather than composing Tailwind utilities.

**All copy/links live in data modules** — `src/data.js` for the homepage + shared chrome (nav links, releases, footer; footer column entries are `string` for dead `#` links or `{label, href}` for internal routes), `src/useCasesData.js` for the use-cases page (hero copy, filter vocabularies, and the 90-card dataset scraped from the live claude.com grid). Components are presentational; edit content there, not in JSX.

**Routing** (`src/Router.jsx`): a minimal pathname router — `RouterProvider` tracks `location.pathname` via popstate, `RouteLink`/`navigate()` pushState and re-dispatch popstate. Two routes in `App.jsx` (`/` → HomePage, `/use-cases` → UseCasesPage; unknown paths render Home) with per-route `document.title`. Deep links work because `netlify.toml` rewrites `/* → /index.html` and Vite's dev server does the same fallback — don't add a router dependency for this.

**Page composition** is flat: `App.jsx` renders Nav → `<main>` (routed page) → Footer; `HomePage.jsx` renders Hero → AnnouncementCards → LatestReleases → Statement → LogoBanner → CtaSection. Shared inline SVGs are in `src/components/Icons.jsx`. The CTA band and the footer share the same dark surface and sit flush: the page-end margin lives on `.cta` (not `.footer`), and `.footer`'s hairline top border is the seam between them. The use-cases page ends on the light canvas instead, so `.uc-layout` carries its own pre-footer padding.

**Use-cases page** (`src/components/UseCases*.jsx`, styles under `.uc-` in `index.css`): `UseCasesExplorer` owns all state (per-group `Set` selections, query, grid/list view, page) and derives the visible slice with `useMemo` — filters are instant-apply checkboxes (OR within a group, AND across groups and with search over title+description); every filter/search change resets to page 1. `PAGE_SIZE` 15 (3×5, like the reference); pagination is "View more"/"Previous" + a "n / N" indicator, replacing the slice (reference semantics), scrolling back to the top of `.uc-layout`. The card grid's thumbnails are CSS kraft-gradient variants (`.uc-card__media--1..4`, assigned by index) under the same grain overlay as the announcement cards. List view is a CSS-grid table (AUTHOR/CATEGORY/MODEL/FEATURES; the FEATURES column drops out ≤1280px where the main column can't fit it) whose cells carry sr-only label spans — read by AT at every width, shown inline when rows stack ≤767px. Whole-card clicks come from a stretched link (`.uc-card__link::after`); card links point at the real claude.com detail pages. "Browser Use" appears in card data but is deliberately absent from the Features filter vocabulary — same as the reference.

**Scroll animation** (`src/components/AnnouncementCards.jsx`): a GSAP ScrollTrigger-scrubbed tween drives a single `--expand` CSS variable from 0→1; CSS in `index.css` interpolates the pair's `max-width` **and `height`** and each card's `border-radius` against it (inset 1272px × min(720px, 100svh) row → full-bleed 100svh viewport takeover, with the gap between the two cards constant throughout). The trigger uses **absolute scroll offsets**, not section-relative ones: one mouse-wheel notch ≈ 100px, and `start: 200, end: 500` plays the takeover across wheel notches 3–5; the tween's `sine.inOut` ease gives the middle notch (the 4th) half the expansion, and `scrub: 0.4` smooths between wheel events. Absolute offsets are layout-independent, so no refresh-after-fonts plumbing is needed. The section reserves the full takeover height in layout (`min-height: 100svh`, cards top-aligned — the reserved slack sits below the cards, keeping the hero→cards gap tight, and is consumed as they grow), so the height animation never reflows content below. **The card copy floats**: the eyebrow + CTA live in `.feature-overlay` — a sibling layer absolutely positioned over the section, frozen at the pair's initial inset geometry (`width: min(1272px, 100% - 48px)`, height = initial card height; offsets resolve against the section's padding edge, which the animated `padding-inline` never moves) — so the text keeps normal scroll displacement while the cards expand underneath it — the copy is centred in each card's inset footprint (`pointer-events: none` on the layer, `auto` restored on the copy elements). Key invariants: `--expand` is scrub-pinned to the real scroll position so it can never stick expanded; `prefers-reduced-motion` drops the height reservation and leaves the pair inset (static layout, matching the overlay's frozen geometry) and freezes the cards' CSS placeholder-media drift. Tune the feel via `start`/`end`/`scrub`/`ease` in the tween config. Each card's "video" is CSS-only (`.vcard__media` gradients + grain); drop a `<video autoplay muted loop playsinline>` inside `.vcard__media` to use real footage.

**Hero word reveal** (`src/components/WordReveal.jsx`): splits the headline into per-word spans at render time (recursing into inline anchors), each with a random 100–500ms `transition-delay`; an IntersectionObserver (threshold 0.2) adds `.is-revealed` and CSS transitions (800ms expo-out, 24px rise, defined in `index.css`) do the animation. The intact sentence is kept in a `.u-sr-only` copy; the split copy is `aria-hidden`.

**Logo banner** (`src/components/LogoBanner.jsx`): a pure-CSS "trusted by" marquee between Statement and the CTA band (it stays on the light canvas, ahead of the dark CTA+footer block). The `width: max-content` track holds two identical `.logos__set` halves, each repeating the six brand marks (`src/components/LogoMarks.jsx`, content in `data.js` `logoBanner`) `SET_REPEATS` (6) times; `logos-scroll` slides it `translateX(-50%)` (72s linear infinite ≈ 61px/s) for a seamless loop. Two invariants: each half must outspan the viewport (one 6-logo run is only ~735px — unrepeated, the strip empties from the right mid-cycle), and each `.logos__set` carries the inter-set gap as `padding-right`, so keep the gap off `.logos__track` or the wrap point jumps. Hover pauses via `animation-play-state`; `prefers-reduced-motion` leaves a static row. The whole marquee is `aria-hidden` (it's decorative under repetition); a `.u-sr-only` list before it names the companies once for AT. AIMA is a typeset wordmark (no clean vector mark exists); the other marks' viewBoxes are cropped to content bounds so per-mark CSS heights set visual weight.

**Fonts are deliberate substitutes**: Source Serif 4 (headlines) and Inter (UI) from Google Fonts (loaded in `index.html`), standing in for the proprietary Anthropic Serif/Sans. The announcement cards' CSS-animated kraft gradients + grain overlay stand in for non-redistributable video footage.
